# [요구사항 정의서] 글로벌 랭킹 시스템: 명예의 퇴근 명부 (Honor Roll of Leaving)

## 1. 개요 (Overview)
본 시스템은 플레이어가 게임을 최종 클리어했을 때의 기록(**칼퇴 시간**)을 서버 기반의 저장소에 기록하고, 실시간으로 순위를 산출하여 게임 내 UI로 제공하는 것을 목적으로 한다. "누가 가장 빨리, 그리고 많은 수당을 챙겨 퇴근했는가"를 겨루는 글로벌 경쟁 요소를 도입한다.

## 2. 상세 요구사항 (Detailed Requirements)

### 2.1 데이터 모델 및 저장 (Data & Storage)
*   **저장 항목**: 
    - `rank_id` (PK): 고유 식별자.
    - `player_name` (String): 사용자 이름.
    - `clear_time` (Float): 칼퇴 시간 (초 단위).
    - `total_overtime_pay` (Int): 누적 야근수당.
    - `checksum` (String): 데이터 변조 방지용 해시값.
    - `created_at` (Timestamp): 기록 달성 일시.
*   **저장소 방식**: Node.js(Express) 기반 REST API를 통해 접근하는 서버 내 데이터베이스(SQLite 권장).
*   **인터페이스**: 
    - `GET /api/rankings`: 상위 10위 및 플레이어의 현재 순위 조회.
    - `POST /api/rankings`: 새로운 기록 제출.

### 2.2 랭킹 산출 및 저장 로직 (Ranking Logic)
*   **트리거 조건**: 최종 보스 처치 후 게임 클리어 이벤트 발생 시.
*   **저장 조건**:
    1. 현재 기록이 전체 TOP 10 이내이거나 플레이어의 개인 최고 기록(Personal Best)을 갱신했을 때.
    2. 사용자 승인: 저장 조건 충족 시 팝업을 통해 '퇴근 기록을 제출하시겠습니까?' 확인 후 'Yes' 응답 시 최종 저장.
*   **정렬 기준 (Priority)**:
    1.  **칼퇴 시간 (clear_time)**: 짧을수록 높은 순위 (오름차순).
    2.  **야근수당 (total_overtime_pay)**: 시간이 동일할 경우, 수당이 많을수록 상위 (내림차순).
    3.  **달성 시간 (created_at)**: 위 지표가 모두 동일할 경우, 먼저 달성한 유저를 상위에 노출.

### 2.3 플레이어 식별 (Identification)
*   **플레이어명**: 세션 시작 시 입력한 이름을 사용하며, 서버 전송 전 유효성 검사(비속어 필터링 등)를 수행한다.

### 2.4 UI/UX 및 출력 (Interface)
*   **구현 방식**: 게임 Canvas 위의 **HTML DOM Overlay**를 사용하여 가독성과 스크롤 편의성을 확보한다.
*   **실시간 조회 (K-Key)**: 인게임 중 'K' 키 입력 시 화면 중앙에 '명예의 퇴근 명부' 팝업 노출.
*   **구성 요소**:
    - **TOP 10 리스트**: 1위부터 10위까지의 이름, 칼퇴 시간, 야근수당 표시.
    - **내 기록 (My Status)**: 리스트 하단에 플레이어 본인의 현재 순위와 최고 기록을 별도 표시.
*   **상태별 노출**:
    - **사망 시**: 저장 절차 없이 현재 랭킹 리스트만 확인 가능.
    - **클리어 시**: 기록 경신 시 저장 유도 팝업 -> 저장 후 랭킹 보드 자동 노출.

## 3. 기술적 제약 사항 및 보안 (Technical Constraints & Security)

### 3.1 보안 및 데이터 무결성
*   **서버 전용 체크섬**: HMAC-SHA256 체크섬은 **서버에서만 생성**하여 DB에 저장한다. 클라이언트에는 시크릿 키를 두지 않으며, 데이터(`player_name`, `clear_time`, `total_overtime_pay`)만 전송한다.
*   **환경변수 필수**: `RANKING_SECRET_KEY` 환경변수가 설정되지 않으면 서버가 시작을 거부한다.
*   **입력 검증**: `player_name`은 1~10자 제한, HTML 태그 포함 시 거부(XSS 방지). `clear_time` 30초 미만은 자동 거부.
*   **CSP**: Content Security Policy를 통해 인라인 스크립트 실행 및 외부 리소스 로드를 차단한다.
*   **XSS 방어**: 서버에서 수신한 데이터를 클라이언트에서 렌더링할 때 `escapeHtml()` 함수로 이스케이프한다.

### 3.2 예외 처리 (Exception Handling)
*   **오프라인 대응**: 서버 연결 실패 시 기록을 `localStorage`의 큐(Queue)에 임시 저장(Buffering)한다. 
*   **재시도 로직**: 다음 게임 실행 시 또는 네트워크 상태가 복구되었을 때 백그라운드에서 전송을 재시도한다.
*   **동시성 제어**: 서버는 DB 트랜잭션을 사용하여 다수의 사용자가 동시에 기록을 제출할 때 데이터가 유실되거나 꼬이지 않도록 관리한다.
